## 朝：デイリーノートにタスクを書く

朝にやる最初の作業は、**今日のタスクを書き出すこと**です。  
まずは紙に軽くメモし、次に Obsidian の デイリーノート へ転記する流れが自然で続けやすい方法です。

---

## 1. 今日やることを紙のノートにメモする（推奨）

システムとは連動しませんが、はじめに今日やることを紙にメモすることを推奨します。  
ポモドーロ・テクニック(*)の表記方法で以下例のように記します。

![１日のタスクリスト](images/todo_note.png)

- 朝一に今日１日のタスクリストを記入
- 右側にポモドーロ予定数(〇)、優先度を記入
- 日中、振り返りは赤字てメモを記入、右側の〇の隣に実績(✔)を記入

メモがまとまったら、Obsidian へ転記します。

### (*)ポモドーロ・テクニックとは、

時間を区切って一つの作業に集中する仕組みで、ポモドーロ１回(上記〇)につき、以下のステップを実行します。

-  達成しようとするタスクを選ぶ  
-  タイマーで25分を設定する  
-  タイマーが鳴るまでタスクに集中する  
-  少し休憩する（5分、15分）  

[ポモドーロ・テクニックとは？](https://www.lifehacker.jp/article/140714pomodoro/)　
日本版 Lifehacker による解説。

## 2. Obsidian へタスクリストを書く

![今日のデイリーノート](images/step1_create_daily_note.png)

画面左側のアイコンから **「今日のデイリーノートを開く」** を選択すると、  
`_journal/yyyy-mm-dd.md` が自動生成されて開きます。

この デイリーノート は以下の用途で使用します：

- **朝**：今日の予定タスクの記入  
- **夜**：ptune-log による振り返りレポートの作成

日次の作業記録はすべてこの デイリーノート に集約されます。　　

「今日の予定タスク」セクションの下に移動して、以下の手順でタスクリストを作成します。

![今日の予定タスク入力](images/step1_update_today_tasks.png)

* 親タスク（今日の主要作業）を書き、必要に応じてサブタスク（事前調査・試作・準備など）に分解する
* 記法の使い方

  | 記法          | 意味                               |
  | ----------- | -------------------------------- |
  | 🍅x1 / 🍅x2 | 予定ポモドーロ数。`::` を入力すると候補がサジェストされます |
  | 🚫          | 今日だけの雑タスク・即時タスク（記録を残さなくてよいもの）    |
  | `<朝>`       | 時間帯の分類に使用。任意で利用可能                |

## 3. チュートリアル用のタスク例（コピーして試せます）

以下の例をそのまま デイリーノート に貼り付けて試せます。

```markdown
## ✅ 今日の予定タスク（手動で追記OK）

- [ ] <朝>くすり🚫
- [ ] Obsidianセットアップ検証
	- [ ] 仕様検討🍅x1
	- [ ] プロトタイプ🍅x2
- [ ] ユーザテストポータルページ作成
	- [ ] 現状調査🍅x2
	- [ ] プロトタイプ🍅x2
```

## 4. 次のステップ：Google Tasks へ同期する

ptune と ptune-log は **Google Tasks を介して連携**するため、
デイリーノート に書いたタスクを Google Tasks に同期する必要があります。

次の手順で同期します。

![エクスポート](images/step1_export_command.png)

1. Obsidian の **コマンドパレット(Ctrl+P)**を開く
2. **「デイリーノートからタスクのエクスポート」** を実行
   → ptune-log が デイリーノート のタスクを Google Tasks に送信します
3. スマホアプリ **ptune** を開く
4. 右上の **同期画面** を開き **「タスクを取り込む」** をタップ
   → Obsidian のタスクがスマホ側に反映され、タイマーで実行できます

![エクスポート](images/step1_ptune_import.png)

初回の実行の場合、**Google OAuth 認証** が表示されます。
