## 機能一覧

朝・日中・夜・セットアップで使うコマンドと補助機能をまとめます。

## 朝: 今日の予定タスクの作成

### `今日のデイリーノートを開く`

- Ctrl+P でコマンドパレットを開き、「今日のデイリーノートを開く」を選択します
- または、画面左側のコマンドバーの「今日のデイリーノートを開く」をクリックします
- 開いたデイリーノートの `今日の予定タスク` の下に、その日のタスクリストを記述します

### `今日の予定を Pull`

- 前日の未完了タスクなど、Google Tasks のタスクを事前に取り込む場合に実行します
- Ctrl+P でコマンドパレットを開き、「pull」で検索して `今日の予定を Pull` を実行します
- 初回の `Push` の前と後でモードが変わります
- 初回は `planning` モードとして未完了タスクを取り込みます
- その後は `working` モードとして完了済みタスクも含めて差分を取り込みます

### `今日の予定を Push`

- デイリーノートで予定タスクを編集したあと、Google Tasks にタスクを反映するときに実行します
- Ctrl+P でコマンドパレットを開き、「push」で検索して `今日の予定を Push` を実行します
- 初回の `Push` の前と後でモードが変わります
- 初回は `planning` モードとして洗い替えで同期し、Google Tasks 側の当日タスクを入れ替えます
- 残したい既存タスクがある場合は、先に `Pull` で取り込んでから `Push` します
- その後は `working` モードとして、変更や追加があったタスクを差分で同期します

## 日中: 作業記録を残す

### `プロジェクトフォルダ作成`

- ファイルエクスプローラで `_project` フォルダを右クリックして実行します
- 新しい作業テーマごとに、記録用のプロジェクトフォルダを作成します
- 作成後は対象のプロジェクトフォルダ下にノートを記録します

### `ノート作成`

- ファイルエクスプローラで既存のプロジェクトフォルダを右クリックして実行します

### `ノート要約を生成`

- 対象ノートを開いた状態で、Ctrl+P から `ノート要約を生成` を実行します
- または、Markdown ノートを右クリックして `ノート要約` を実行します
- ノート内容をもとに、振り返りで使いやすい要約文を作成または編集します

## 夜: 振り返る

### `今日の振り返り Review`

- 夜の終了時に、その日の振り返り内容をデイリーノートへ出力します
- Ctrl+P でコマンドパレットを開き、「review」で検索して `今日の振り返り Review` を実行します
- Google Tasks 側の実績と、その日に作成したノートをもとにレポートを作成します

## セットアップ・認証

### `セットアップ確認`

- 初回導入時に、必要な設定がそろっているか確認するときに実行します
- Ctrl+P でコマンドパレットを開き、「セットアップ確認」を実行します
- 本確認画面で、`Daily Notes` `PtuneSync` ノート関連ディレクトリなど、利用開始に必要な項目を確認します

### `ログイン認証`

- Google Tasks 連携を使うために、認証を開始するときに実行します
- Ctrl+P でコマンドパレットを開き、「ログイン認証」を実行します
- 初回セットアップ時や、認証が切れたときに使います

### `認証ステータス`

- Ctrl+P でコマンドパレットを開き、「認証ステータス」を実行します
- 認証済みか未認証かを確認できます
