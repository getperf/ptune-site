## ③ 日中: ノートに作業記録を残す

このステップでは、日中の作業内容を **あとで見返して判断材料になる形で残す方法** を説明します。

## 記録の考え方

ノートには会話ログや実行結果を何でも貼るのではなく、**要点を整理して残す** ことを推奨します。

残すとよい内容:

- 何を確認したか
- どの案を採用したか
- 採用理由は何か
- 実行したコマンドや修正内容は何か
- 次に残った課題は何か

残しすぎない方がよい内容:

- AIツールの会話全文  
- 捨てた案の断片
- パスワード、API キー、認証情報、個人情報

## AI ツールの記録例

AI ツールは、会話全文を保存するより、採用した結論や次の行動が分かる形で残します。

例:

- ChatGPT の回答から採用した箇所だけを貼る
- チャット全体ではなく **Copy Final Response** を使い、必要部分だけを転記する
- 最終提案、実行結果の要点、次のアクションだけを残す

## ノートに書く例

```markdown
## ptune-task 利用手順見直し

- usage 章に旧名称が残っていることを確認
- 優先改訂は index / overview / step1 / step5
- 振り返りはユーザ主体、AI は補助に寄せる方針を採用

### 実行内容
- docs の usage 章を確認
- 改訂方針を整理

### 次にやること
- step5 の振り返り説明を書き換える
```

## 次のステップ

作業を一段落させたら、ノートを軽く整理して次の作業に進みます。
